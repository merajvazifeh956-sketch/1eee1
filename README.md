## 🚀 نحوه استفاده (گام‌به‌گام)

### مرحله ۱: فورک کردن ریپو
1. وارد گیت‌هاب شوید و این ریپو را **Fork** کنید.

1. روی دکمه **Code** کلیک کنید.

### مرحله ۴: تنظیم پورت (بسیار مهم)
- در محیط Codespace، تب **PORTS** را باز کنید.
- پورت **443** را پیدا کنید.
- روی آن راست کلیک کرده و **Visibility** > **Public** را انتخاب کنید.

### مرحله ۵: اتصال

## 📖 راهنمای استفاده (Usage)

### 🔗 دریافت و استفاده از لینک
بعد از اجرای کداسپیس، لینک VLESS به صورت خودکار در ترمینال نمایش داده می‌شود. کافی است:
1. لینک را کامل کپی کنید (از `vless://` تا آخر)
2. آن را در اپلیکیشن پروکسی خود وارد کنید (بیشتر اپلیکیشن‌ها با کلیک روی دکمه `+` یا `Import from clipboard` لینک را تشخیص می‌دهند)

### 🔄 نمایش مجدد لینک
اگر به هر دلیلی لینک را گم کردید یا نیاز به کپی مجدد دارید، در ترمینال کداسپیس دستور زیر را اجرا کنید:
```bash
show-link.sh
```

### 📋 مشاهده لاگ‌های Xray
برای مشاهده لاگ‌های سرور و اطمینان از عملکرد صحیح، دستور زیر را در ترمینال وارد کنید:
```bash
tmux attach -t nikvpn
```
برای خروج از نمای لاگ (بدون توقف سرویس)، کلیدهای `Ctrl+B` و سپس `D` را بزنید.

## ⏰ تنظیم تایم‌اوت کداسپیس (برای جلوگیری از خاموشی)

GitHub Codespaces به طور پیش‌فرض بعد از **30 دقیقه** عدم فعالیت، سرور را خاموش می‌کند. برای افزایش این زمان:

### روش ۱: تغییر تنظیمات از طریق گیت‌هاب
1. وارد [github.com/settings/codespaces](https://github.com/settings/codespaces) شوید.
2. در بخش **"Default idle timeout"** گزینه مورد نظر را انتخاب کنید:
   - **15 minutes** (پیش‌فرض)
   - **30 minutes**
   - **60 minutes**
   - **120 minutes**
   - **240 minutes** (بیشترین مقدار ممکن)
3. روی **Save** کلیک کنید.

> ⚠️ توجه: حداکثر تایم‌اوت قابل تنظیم **240 دقیقه (۴ ساعت)** است.

### روش ۲: روش پایدار و همیشگی (اجباری برای ماندگاری نامحدود)

با تجربه‌ای که داشتم، حتی بعد از تغییر تنظیمات تایم‌اوت، باز هم پنجره Keepalive ظاهر می‌شد و اتصال قطع می‌شد. راه‌حلی که برای من جواب داد و کداسپیس تا 240 دقیقه روشن ماند:

**اجرای دستور `tmux attach -t nikvpn` در ترمینال**

وقتی این دستور را اجرا می‌کنید:
1. وارد جلسه tmux می‌شوید که Xray در آن در حال اجراست.
2. تا زمانی که این جلسه **متصل (attached)** است، کداسپیس خاموش نمی‌شود.
3. می‌توانید پنجره ترمینال را باز بگذارید یا آن را minimize کنید.

**برای خروج از جلسه tmux (بدون توقف سرویس):**
- کلیدهای `Ctrl+B` و سپس `D` را بزنید.

> 💡 **نکته طلایی:** اگر می‌خواهید کداسپیس برای مدت طولانی (مثلاً چند ساعت) روشن بماند، کافی است یک ترمینال باز کنید، دستور `tmux attach -t nikvpn` را اجرا کنید و ترمینال را باز بگذارید. دیگر نیازی به نگرانی از خاموشی نیست!

## 📱 کلاینت‌های سازگار (Supported Clients)

پروکسی NikVPN با اکثر کلاینت‌های محبوب که از پروتکل VLESS پشتیبانی می‌کنند، سازگار است:

| پلتفرم | کلاینت‌های پیشنهادی | لینک دانلود |
|--------|---------------------|-------------|
| **اندروید** | v2rayNG، Nekobox، Hiddify | [v2rayNG](https://github.com/2dust/v2rayNG/releases) / [Nekobox](https://github.com/MatsuriDayo/NekoBoxForAndroid/releases) |
| **ویندوز** | v2rayN، Nekoray، Hiddify-Next | [v2rayN](https://github.com/2dust/v2rayN/releases) / [Nekoray](https://github.com/MatsuriDayo/nekoray/releases) |
| **iOS (iPhone/iPad)** | Streisand، FoXray، V2Box | [Streisand](https://apps.apple.com/app/streisand/id1635313760) (App Store) |
| **لینوکس** | Nekoray، Xray-core، Hiddify-Next | [Nekoray](https://github.com/MatsuriDayo/nekoray/releases) |

> ⚠️ **توجه:** در برخی کلاینت‌ها ممکن است نیاز باشد تنظیمات زیر را به صورت دستی اعمال کنید:
> - پروتکل: `VLESS`
> - ترانسپورت (Network): `xHTTP`
> - mode: `packet-up`
> - امنیت (Security): `TLS` (با allowInsecure=1)

## 🧼 گزینه‌های آی‌پی تمیز (Clean IP Options)

آی‌پی پیش‌فرض استفاده شده در لینک `94.130.50.12` است. اگر این آی‌پی در شبکه شما (به خصوص در ایران) مسدود یا با اختلال مواجه است، می‌توانید آن را در لینک VLESS با یکی از آی‌پی‌های زیر جایگزین کنید:

### 🔽 آی‌پی‌های جایگزین (تمیز و تست شده)

| آی‌پی | وضعیت تقریبی |
|-------|---------------|
| `63.141.252.203` | ✅ مناسب برای ایرانسل |
| `50.7.5.83` | ✅ مناسب برای همراه اول |
| `94.130.50.12` | ⭐ پیش‌فرض (همراه اول/ایرانسل) |

### 📝 نحوه جایگزینی آی‌پی
لینک اصلی شما به این شکل است:
```
vless://UUID@94.130.50.12:443?encryption=...
```
برای تعویض آی‌پی، کافی است `94.130.50.12` را با آی‌پی جدید عوض کنید:
```
vless://UUID@63.141.252.203:443?encryption=...
```

> 💡 **نکته حرفه‌ای:** اگر هیچکدام از آی‌پی‌ها کار نکرد، می‌توانید از سرویس‌های پیداکننده آی‌پی تمیز کلودفلیر (مثل `CloudflareScanner`) استفاده کنید و یک آی‌پی جدید با پورت 443 پیدا کنید.

## 🆘 عیب‌یابی (Troubleshooting)

| مشکل | راه حل |
|------|--------|
| اتصال برقرار نمی‌شود | 1. پورت 443 را در تب PORTS چک کنید (باید Public باشد)<br>2. آی‌پی را با Clean IP جایگزین کنید<br>3. در کلاینت، `allowInsecure` را روی `true` یا `1` تنظیم کنید |
| سرعت پایین است | آی‌پی دیگری از لیست Clean IPها امتحان کنید یا کداسپیس را ری‌استارت کنید |
| لینک در ترمینال نمایش داده نمی‌شود | دستور `show-link.sh` را به صورت دستی اجرا کنید |
| خطای "UUID not found" | کداسپیس را حذف و دوباره بسازید |
| کداسپیس زود خاموش می‌شود | دستور `tmux attach -t nikvpn` را اجرا کنید و ترمینال را باز بگذارید |
